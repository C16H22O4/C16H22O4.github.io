# 蓝桥杯 Finder 使用手册

蓝桥杯 Finder 是一个纯前端的蓝桥杯软件赛获奖数据查询工具。项目通过静态页面和分片 JSON 数据实现查询能力，可以直接部署到 GitHub Pages，也可以在本地用一个简单的静态 HTTP 服务预览。

## 功能概览

- 查询选手获奖汇总，支持按姓名、拼音首字母和学校筛选。
- 查询比赛名单，支持按省赛/国赛、届次、语言、组别、省份、学校和姓名筛选。
- 查看学校获奖排行和学校详情。
- 查看选手个人获奖明细。
- 首页展示数据总览、选手排行和学校排行。

## 项目结构

```text
.
├── index.html          # 主页面
├── 404.html            # GitHub Pages 前端路由兜底页面
├── app.js              # 页面交互、路由和数据查询逻辑
├── styles.css          # 页面样式
├── logo.png            # 网站图标
└── data/               # 静态分片数据
    ├── manifest.json   # 数据版本、分片大小和统计信息
    ├── meta.json       # 数据总览
    ├── home.json       # 首页排行榜数据
    ├── options.json    # 比赛筛选选项
    ├── dict.json       # 学校、省份、科目等字典
    ├── people/         # 选手汇总分片
    ├── details/        # 选手明细分片
    ├── search/         # 选手搜索索引分片
    ├── schools/        # 学校数据和学校详情分片
    ├── school_rows/    # 学校到选手行号的映射分片
    ├── school_search_ids/
    ├── person_lookup/
    └── competitions/   # 比赛名单分片
```

## 本地运行

不要直接双击打开 `index.html`。项目里大量使用 `fetch("/data/...")` 读取静态 JSON，直接用文件方式打开会受到浏览器限制，推荐使用本地 HTTP 服务。

进入项目目录：

```powershell
cd C:\Users\fangchengye\My\C16H22O4.github.io
```

启动本地服务：

```powershell
python -m http.server 8000
```

然后在浏览器打开：

```text
http://localhost:8000
```

关闭服务时，在运行服务的 PowerShell 窗口按 `Ctrl + C`。如果已经直接关闭了 PowerShell，服务通常也会一起结束。

## 使用方式

### 首页

打开网站后默认进入首页。首页会展示当前数据规模、选手排行、学校排行和常用入口。

### 查询选手

点击顶部导航的「选手」，可以：

- 在姓名输入框中输入中文姓名。
- 输入拼音或拼音首字母进行检索。
- 输入学校名称进一步筛选。
- 点击表格中的选手姓名查看个人获奖明细。

### 查询比赛名单

点击顶部导航的「比赛」，可以：

- 选择省赛或国赛。
- 查询省赛时需要选择省份。
- 按届次、语言和组别筛选。
- 按学校或选手姓名关键字过滤结果。

### 查询学校

点击顶部导航的「学校」，可以：

- 查看学校获奖排行。
- 输入学校名称筛选。
- 点击学校名称查看该学校的获奖明细。

## 数据说明

项目数据存放在 `data/` 目录中，采用分片 JSON 的方式保存。这样可以避免浏览器一次性加载完整数据，提升首屏和查询体验。

当前数据概览可以查看：

```text
data/manifest.json
data/meta.json
```

如果只修改页面样式或交互，通常只需要改 `index.html`、`styles.css` 和 `app.js`。如果要更新获奖数据，需要重新生成 `data/` 目录下的分片文件，并同步更新 `manifest.json`、`meta.json`、`home.json` 和相关索引。

## 部署说明

这个项目适合直接部署到 GitHub Pages。推送到仓库后，在 GitHub 仓库设置中启用 Pages，并选择对应分支作为发布源即可。

由于项目使用前端路由，`404.html` 用来处理刷新详情页或直接访问子路径时的兜底。部署时请保留 `404.html`。

## 常见问题

### 页面一直显示加载中

通常是本地打开方式不对。请确认是通过 `http://localhost:8000` 访问，而不是直接双击 `index.html`。

### 查询很慢

首次访问需要加载部分字典和索引数据，数据量较大时会稍慢。后续查询通常会因为浏览器缓存而变快。

### 中文显示乱码

页面文件使用 UTF-8 编码。浏览器正常打开时应显示中文。如果在 PowerShell 中查看文件出现乱码，通常是终端编码显示问题，不代表文件损坏。

### 8000 端口被占用

可以换一个端口启动：

```powershell
python -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```
